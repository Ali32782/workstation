# Subdomain Plan

Two active platform domains on a single host, isolated by Keycloak realms +
subdomains. **All Kineo360 services live under a tenant subtree** — there are
no "shared" services at the apex. MedTheris, as the product operator, owns
the services that every practice consumes.

| Domain             | Purpose                                                 |
|--------------------|---------------------------------------------------------|
| `corehub.io`       | Corehub Technologies — dev team (internal)              |
| **`kineo360.work`**| **Kineo360 product platform — every service under a tenant** |
| `medtheris.com`    | (optional) MedTheris GmbH mail domain                   |

---

## Naming convention for Kineo360

```
<tenant>.kineo360.work              — tenant root (landing / dashboard)
<service>.<tenant>.kineo360.work    — one service of that tenant
```

Tenants:

| Tenant       | Role                                              |
|--------------|---------------------------------------------------|
| `medtheris`  | MedTheris GmbH — operator, hosts shared services  |
| `<slug>`     | Paying practice, e.g. `mueller`, `zurich-nord`    |

Services:

| Service    | Purpose                                               |
|------------|-------------------------------------------------------|
| `auth`     | Keycloak login endpoint for the tenant realm          |
| `files`    | Nextcloud for this tenant                             |
| `chat`     | Rocket.Chat workspace for this tenant                 |
| `meet`     | Jitsi Meet (on MedTheris tenant only — shared usage)  |
| `office`   | Collabora Online (on MedTheris tenant only)           |
| `support`  | Zammad helpdesk (on MedTheris tenant only)            |
| `status`   | Uptime Kuma (on MedTheris tenant only, phase 2)       |

Apex (`kineo360.work`) and `www.kineo360.work` are reserved for the public
marketing landing page — no application services there.

## Kineo360 — MedTheris tenant (`medtheris.kineo360.work`)

MedTheris is a tenant **and** the operator. It hosts every service that
practices consume.

| Subdomain                                   | Target Container         | Port |
|---------------------------------------------|--------------------------|------|
| `medtheris.kineo360.work`                   | landing / dashboard      | 80   |
| `auth.medtheris.kineo360.work`              | `keycloak` (realm `medtheris-internal`) | 8080 |
| `files.medtheris.kineo360.work`             | `nextcloud-medtheris`    | 80   |
| `chat.medtheris.kineo360.work`              | `rocketchat`             | 3000 |
| `meet.medtheris.kineo360.work`              | `jitsi-web`              | 80   |
| `office.medtheris.kineo360.work`            | `collabora`              | 9980 |
| `support.medtheris.kineo360.work`           | `zammad-nginx`           | 8080 |
| `status.medtheris.kineo360.work`            | `uptime-kuma`            | 3001 |

Practices embed the shared services directly:

- Video calls from `chat.mueller.kineo360.work` → `meet.medtheris.kineo360.work`
- Support tickets from any tenant → `support.medtheris.kineo360.work`
- Office editing in `files.mueller.kineo360.work` → `office.medtheris.kineo360.work`

## Kineo360 — practice tenants (generated by `onboard-practice.sh`)

For every onboarded practice `<slug>`:

| Subdomain                             | Target Container       | Port |
|---------------------------------------|------------------------|------|
| `<slug>.kineo360.work`                | landing / dashboard    | 80   |
| `auth.<slug>.kineo360.work`           | `keycloak` (realm `practice-<slug>`) | 8080 |
| `files.<slug>.kineo360.work`          | `nc-<slug>`            | 80   |
| `chat.<slug>.kineo360.work`           | `rocketchat` (workspace) | 3000 |

Example (`<slug> = mueller`):

```
mueller.kineo360.work
auth.mueller.kineo360.work
files.mueller.kineo360.work
chat.mueller.kineo360.work
```

Practices **do not get their own** `meet.`, `office.`, `support.`, `status.`
— they use the MedTheris-operated ones.

### Wildcard DNS — strongly recommended

```
*.kineo360.work.         A    <server-ipv4>
*.*.kineo360.work.       A    <server-ipv4>   ← required for *.medtheris. and *.<slug>.
```

Pair with a Let's Encrypt wildcard cert via **DNS-01** in NPM:

- Cert A: `*.kineo360.work` + `kineo360.work`
- Cert B: `*.*.kineo360.work`

If the registrar of `kineo360.work` cannot serve `*.*.`, move DNS to a
provider that can (Cloudflare works) — or add per-tenant A records during
onboarding (the script prints them).

## Admin access — **never public**

```bash
# Portainer (port 9000 bound to 127.0.0.1):
ssh -L 9000:localhost:9000 deploy@<host>
# → http://localhost:9000

# NPM admin UI (port 81 bound to 127.0.0.1):
ssh -L 81:localhost:81 deploy@<host>
# → http://localhost:81
```
