# onedoc-scraper

Daily-cycle scraper for Kineo's OneDoc slot availability + downstream
reporting/forecasting/email pipeline. Runs as a systemd service on the
Hetzner host (no Docker — Playwright headless Chromium is finicky in
containers and the data volume is small enough that a single process is
fine).

## Layout

| File                         | Role                                                              |
| ---------------------------- | ----------------------------------------------------------------- |
| `scraper_api.py`             | Hits OneDoc's slot API for every (standort, therapist) tuple.    |
| `scraper.py`                 | Legacy Playwright fallback (only used if the API path fails).    |
| `discover_all_v2.py`         | Crawls each Kineo OneDoc practice page, parses prof_id/cal_id.   |
| `discover_diff.py`           | Daily-cron diff: alerts on new/removed therapists. Added 2026-05-03. |
| `discovery.py`               | SQLite layer for `standorte` / `therapeuten` / `discovery_log`.   |
| `praxen_config.json`         | Current snapshot (regenerated by `discover_all_v2`).             |
| `adaptive_scheduler.py`      | Master loop — invokes scraper_api, reporting, train_forecast.    |
| `reporting.py`               | Builds daily/weekly status html.                                  |
| `email_report.py`            | SMTP fan-out (uses `EMAIL_*` env vars).                           |
| `teams_report.py`            | Microsoft-Teams Incoming-Webhook variant of the same report.      |
| `forecasting.py` / `train_forecast_model.py` | Demand forecasting from historic snapshots.       |
| `install_update*.py`         | One-shot installers (kept for parity with the Mac dev box).       |

`update_dashboard-2.py` … `update_dashboard-7.py` are legacy snapshots
of historic dashboard generators. Kept to preserve history; safe to
delete once nobody references them.

## Deploy

The host expects this directory at `/opt/corelab/onedoc_scraper/` (note
the underscore — historic). To push local changes:

```bash
rsync -az --delete \
  --exclude=.venv --exclude=__pycache__ --exclude='*.pyc' \
  --exclude=.env --exclude='*.log' --exclude=logs \
  --exclude=screenshots --exclude=models \
  --exclude='*.db' --exclude='*.sqlite*' \
  services/onedoc-scraper/ \
  medtheris-corelab:/opt/corelab/onedoc_scraper/
```

Then restart the affected systemd unit:

```bash
ssh medtheris-corelab-root 'systemctl restart onedoc-scheduler.service'
```

## Two systemd units

```bash
ssh medtheris-corelab 'systemctl list-timers onedoc-* --all'
```

- `onedoc-scheduler.service` — adaptive-scheduler loop (always on).
- `onedoc-discovery.timer` — daily 06:30 + 10 min jitter, runs `discover_diff.py`
  to catch new/removed therapists. Posts to Rocket.Chat if
  `ROCKETCHAT_DISCOVERY_WEBHOOK` is set in `.env`.

## Required env vars

In `/opt/corelab/onedoc_scraper/.env` (gitignored, server-only):

```
EMAIL_HOST=…           # SMTP for daily report
EMAIL_USER=…
EMAIL_PASSWORD=…
EMAIL_TO=…
TEAMS_WEBHOOK_URL=…    # optional, for teams_report.py
ROCKETCHAT_DISCOVERY_WEBHOOK=…  # optional, for discover_diff.py
```

## SQLite database

`slots.db` (gitignored) is the canonical store. Schema in `discovery.py`:

- `standorte` — Kineo practice sites
- `therapeuten` — therapists, with fail-streak tracking
- `discovery_log` — timeline of "neu/inaktiv" events
- `slot_snapshots` — daily snapshot per (date, standort, therapeut)
- `observations` — slot-counts by week (KW0…KW4)

If you wipe `slots.db`, the next scrape repopulates everything except the
historical observations.
